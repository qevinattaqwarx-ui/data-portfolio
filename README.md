Project Overview

This project is a SQL-based data analysis case study focused on transforming raw, unstructured data into actionable business insights.

The goal is to simulate a real-world data analyst workflow:

Cleaning messy datasets
Structuring data for analysis
Performing exploratory data analysis (EDA)
Extracting insights that support business decisions

This project demonstrates practical analytical thinking rather than just technical SQL execution.

Data Cleaning Process

The dataset was prepared for analysis through structured cleaning steps:

Removed duplicate records to ensure data integrity
Handled missing and null values appropriately
Standardized inconsistent formats (dates, text fields, categories)
Corrected structural inconsistencies in key columns
Validated data accuracy before analysis

This step ensures that all insights are based on reliable and consistent data.

Exploratory Data Analysis (EDA)

The analysis was designed to identify patterns, trends, and business-relevant signals from the dataset.

Key analytical approaches:

Aggregation using GROUP BY for category-level insights
Time-based trend analysis to identify changes over periods
Segmentation analysis to compare performance across groups
Ranking analysis to identify top and bottom contributors
Filtering and drilling down into high-impact data segments
Key Business Insights
A small number of categories/segments contribute disproportionately to overall outcomes (indicating a skewed distribution)
Activity levels vary significantly across time periods, suggesting potential seasonality or external influence factors
Certain segments consistently underperform, indicating possible inefficiencies or structural issues
Top-performing categories show stable and repeatable patterns, making them strong candidates for focus or optimization
Data patterns suggest opportunities for targeted decision-making rather than uniform strategy
